- Issue was in **authStore.js** — it had old logic that defaulted to `http://localhost:3000/api` for dev

## Root Cause Analysis
The issue was **timing & logic flow**:
1. Created `getApiUrl()` helper in authStore
2. But it wasn't being called consistently on every API request
3. Zustand store evaluation wasn't timing it correctly
4. Result: fallback to localhost was always triggered in production

## Solution Applied

### Fix 1: Rewrote getApiUrl() Function
**File:** `apps/worksafeai/web/src/stores/authStore.js`

```javascript
function getApiUrl() {
  // Check env var first
  if (import.meta.env.VITE_API_URL) {
    return import.meta.env.VITE_API_URL;
  }
  
  // Production: detect from current domain
  if (typeof window !== 'undefined' && window.location.hostname !== 'localhost') {
    const hostname = window.location.hostname;
    let apiHostname = hostname.replace('worksafeai.', 'worksafeai-api.');
    apiHostname = apiHostname.replace('superadmin.', 'worksafeai-api.');
    return `https://${apiHostname}/api`;
  }
  
  // Dev fallback
  return 'http://localhost:3000/api';
}
```

Key improvements:
- ✅ Function, not const (re-evaluates on each call)
- ✅ Forces `https://` for production
- ✅ Handles both `worksafeai.` and `superadmin.` domains
- ✅ Called freshly in every login/register method

### Fix 2: Applied Same Fix to Super Admin
**File:** `apps/super-admin/src/stores/authStore.js`
- Same `getApiUrl()` improvement
- Converts `superadmin.elevationaiwork.com` → `worksafeai-api.elevationaiwork.com`
