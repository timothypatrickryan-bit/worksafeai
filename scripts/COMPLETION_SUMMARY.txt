================================================================================
                            HOW IT WORKS
================================================================================

BRIEFING GENERATION (Automatic):
  - Analyzes task category (Design, Infrastructure, Strategy, etc.)
  - Generates category-specific deliverables list
  - Creates milestone timeline (2-4 checkpoints)
  - Defines quality gates (success criteria)
  - Calculates timeline (1-4 hours estimated)
  - Finds related tasks & dependencies

INTERACTIVE APPROVAL:
  - Displays complete briefing with all details
  - Shows deliverables, milestones, quality gates
  - Prompts user: "Approve? (yes/no)"
  - Records approval decision
  - Supports rejection & queuing for later

AGENT EXECUTION:
  - Spawns assigned agent with full briefing
  - Passes execution context & parameters
  - Agent processes task independently
  - Progress tracked via checkpoints
  - Results saved to state

STATE PERSISTENCE:
  - Maintains .mission-control-state.json (main state)
  - Creates .workflow-state.json (workflow tracking)
  - Creates .workflow/task-*.json (per-task execution)
  - Survives application restarts
  - Easy to query and monitor
