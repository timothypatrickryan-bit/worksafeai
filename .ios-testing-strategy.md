## Executive Summary

The iOS Mission Control app is feature-complete with all 5 screens implemented. This phase focuses on:
1. **Component & Integration Testing** - Verify all screens and data flows
2. **E2E Testing** - Complete user workflows (Portfolio → Project → Task → Approval)
3. **Performance Optimization** - Memory, battery, network efficiency
4. **UX Polish** - Visual refinement, animation smoothness, accessibility
5. **Local Tunnel Testing** - Verify iPhone ↔ local machine connectivity

**Success Criteria:**
- All screens load in <2s (iOS 16+, iPhone SE & 15 Pro)
- Battery drain <5% per 30min normal use
- Offline-first working on device
- Real-time sync over WebSocket tunnel
- Zero critical bugs
- WCAG AA accessibility compliance
- Smooth 60 FPS animations

### 2. DATA SYNC & OFFLINE TESTING

**Primary Goal:** Verify offline-first architecture works correctly on device.

#### 2.1 Offline Queue
```
Test Cases:
✓ Create task while offline → queued

✓ Approve task while offline → queued
✓ Queue persists across app restart
✓ Max queue size: 100 actions
```

**Implementation:**

#### 2.2 Real-Time WebSocket
```
Test Cases:

✓ Project status change syncs in <1 second
```

**Implementation:**
- Test with local tunnel (ngrok/Cloudflare)

- Measure WebSocket latency <500ms

#### 2.3 Conflict Resolution

```
Test Cases:
```

### 3. PERFORMANCE OPTIMIZATION

**Primary Goal:** Achieve smooth 60 FPS performance and minimize battery drain.

#### 3.1 Memory Profiling
```
Metrics:
- Baseline memory: <100 MB on launch
- Memory per project: <2 MB (with 10 tasks)
- Peak memory: <250 MB during heavy operations
- No memory leaks: Run for 10 min, check memory stable
```

**Tools:**
- Xcode Memory Debugger
- React Native Profiler
- Measure: `memory.pressure` before/after operations

**Optimization:**
- Image lazy loading in portfolio
- Virtual scroll for 100+ tasks
- Cleanup WebSocket listeners on unmount
- Debounce real-time updates (100ms batches)

#### 3.2 Battery Impact
```
Target:
- Baseline drain: <5% per 30 minutes
- Active use: <8% per 30 minutes
- With WebSocket: <6% per 30 minutes
- Idle (offline): <1% per 30 minutes
```

**Optimization:**
- Batch network requests
- Disable real-time updates when app backgrounded
- Reduce animation FPS when battery low (<20%)
- Compress images: Project photos → max 100KB

#### 3.3 Load Time Optimization
```
Target:
- Cold start: <2 seconds
- App launch to portfolio displayed: <1.5s
- Project detail load: <1s
- Task briefing modal: <500ms
```

**Optimization:**
- Code splitting: Load screens on-demand
- Lazy load project images
- SQLite query indexes on frequently-searched fields
- Pre-fetch next project on portfolio scroll

#### 3.4 Animation Performance
```
Target:
- All animations: 60 FPS minimum
- Kanban drag-drop: Consistent 60 FPS
- Modal transitions: 300ms smooth
- List scroll: No jank on 100+ items
```

**Tools:**
- React Native Performance Monitor
- Xcode Core Animation tool

**Optimization:**
- Use native driver for animations
- Reduce shadow/blur effects on scroll
- GPU-accelerate Kanban drag overlay

### 5. NETWORK & TUNNEL TESTING

**Primary Goal:** Verify iPhone ↔ local machine connectivity over tunnel.

#### 5.1 Local Tunnel Setup
```
Setup:
1. Choose tunnel provider:
   - Option A: ngrok (ngrok http 3000 → get HTTPS URL)
   - Option B: Cloudflare Tunnel (CLI + secure)
   - Option C: LocalTunnel (quick, less reliable)

2. Configure React Native:
   - API_URL env var: https://[tunnel-domain]
   - WebSocket URL: wss://[tunnel-domain]/ws

3. Test tunnel:
   - curl https://[tunnel-domain]/health → 200 OK
   - Check latency: <200ms round trip
```

#### 5.2 Device Testing
```
Test Cases:
✓ iPhone on WiFi: Connect to portfolio

✓ Large response (100+ tasks): Load without OOM
```

**Test Devices:**
